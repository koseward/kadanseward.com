Service Oriented Architecture

Overview

The Service Oriented Architecture (SOA) is a kind of Software Architecture where applications "make use of services available int the network" in order to complete a given task.[1] This allows for a large number of facilities to be used and re-used in order to create different applications. The result is an applications built of smaller parts the communicate and work together.[2]

SOA is often confused with Microservices due to their similar reliance on several, smaller, independent services that make up a greater system. The difference between these is the scope of their implementation. SOA is enterprise wide - meaning a whole company, while Microservices are project wide.[1]

History

SOA first saw use in the late 1990s, when different services saw connections to other systems.[2]

Characteristics

  • Faster project velocity
  • Utilization of legacy software in new markets
  • Improved Company collaboration
