Project Velocity

Overview

Project velocity is a measure of how much work is getting done on a project. It can be measured by adding up completed user stories or requirements and comparing those with the ones remaining. This measurment is used to gauge if the project will be completed on time, and on budget.[1] Project velocity is used by both developers and people higher up to estimate how long the project will take and what some realistic expectations for the project will be.
