Software Architecture

Overview

The Reactive Architecture is a software architecture that focuses four main traits: being responsive, resilient, elastic, and message driven. It was developed as a response to the ever-increasing requirements of modern software. [1] A reactive architecture should be able to function, even if parts of the system are unresponsive. The system must also be able have parts that can function in many different contexts. [2]

History

The reactive manifesto was designed in 2014 in response to the inability of many software architectures to keep up with the demands asked of them. [1]
